## Persistence Across Restarts

The WSL2 IP (`172.17.x.x`) **changes on every WSL2 restart**. Choose one method:

### Option 1: Task Scheduler (Recommended)

1. Open **Task Scheduler** (taskschd.msc)
2. **Create Task:**
   - Name: `WSL2 Port Proxy`
   - Trigger: `At startup`
   - Action: `Start a program`
   - Program: `powershell.exe`
   - Arguments: `-ExecutionPolicy Bypass -File "C:\Users\dominik\.openclaw\workspace\scripts\startup-script.ps1"`
   - Run with highest privileges: ✅
3. **Save**

### Option 2: Shell Startup Folder

1. Press `Win + R`, type: `shell:startup`
2. Copy `startup-script.ps1` to that folder
3. Create shortcut with these properties:
   - Target: `powershell.exe -ExecutionPolicy Bypass -File "C:\Users\dominik\.openclaw\workspace\scripts\startup-script.ps1"`
   - Run as administrator: ✅

## Troubleshooting

**"WSL2 not running" error:**
- Open WSL2 terminal first
- Run: `wsl`

**Port proxy shows old IP:**
- Run: `.\startup-script.ps1` manually
- Or restart the task/scheduled job

**Still can't access:**
- Check Windows Firewall: `Get-NetFirewallRule -DisplayName "OpenClaw Server"`
- Verify router port forward: 8888 → 10.0.0.87:8888
