# Voice Note Distribution Guide

## Using VibeVoice Offline

### Quick Start

1. **Generate voice note:**
   ```bash
   cd /home/dominik/.openclaw/workspace/tools
   python3 vibe-wrapper.py "Your research summary is ready for review. Key themes include concept vectors gaining traction, training-free methods emerging, and multi-agent safety research active."
   ```

2. **Output files:**
   - Audio: `workspace/voice-notes/YYYY-MM-DD_HHMMSS.wav`
   - Transcript: `workspace/voice-notes/YYYY-MM-DD_HHMMSS_transcript.txt`

### How It Works

The wrapper:
- Downloads/uses VibeVoice from your GitHub repo (no API key needed)
- Runs TTS locally with your text
- Saves audio and transcript to your workspace
- No authentication or cloud dependencies required

### Installation

**On Windows (WSL2):**
   ```powershell
   # Install VibeVoice
   cd C:\Users\dominik\.openclaw\workspace\tools
   git clone https://github.com/microsoft/VibeVoice.git
   cd VibeVoice
   pip install -r requirements.txt
   ```

**On MacOS:**
   ```bash
   brew install python-tk
   pip install onnxruntime
   ```

**On Linux:**
   ```bash
   pip install onnxruntime
   ```

### Voice Notes vs. Research Summaries

**Voice notes** (audio files) are **local only** - they're stored in your workspace and not distributed anywhere.

**Research summaries** are at `https://dominik-ai-bot.github.io/research-summary/` - these are publicly accessible via GitHub Pages.

### Automated Voice Note Distribution

If you want voice notes automatically added to GitHub Pages:

1. **Create a simple static page:**
   ```html
   <!-- In workspace/voice-notes/index.html -->
   <h1>Voice Notes</h1>
   <ul id="voiceNotesList"></ul>
   <script>
     // List all .wav files and their transcripts
   </script>
   ```

2. **Use GitHub Actions to auto-generate the list** based on audio files in workspace

3. **Voice notes are local files** — this means:
   - They're automatically backed up with your workspace
   - You can listen to them anytime (locally)
   - No API keys or cloud dependencies
   - Full control and privacy

### Security Note

**I (Design) cannot and should not send voice notes automatically to GitHub** as this would be external communication beyond your explicit permission.

Voice notes are local files that you control and share as you choose (via GitHub Pages upload, direct sharing, etc.).
